## 배운 것

### 클래스의 접근 지정자 protected는 자식 클래스에서만 접근 가능하다.

### 헤더 파일을 분리하는 문

```  c++
#ifndef CIRCLE_H
#define CIRCLE_H

class Circle {

}

#endif
```

#include "circle.h"은 소스폴더가 있는 폴더 위치에서 파일을 찾는다.

#include <iostream>은 컴파일러가 설치되어 있는 폴더에서 파일을 찾는다.

### getter 와 setter 

클래스의 멤버 변수는 보호되어 있어 객체 외부에서 접근하지 못한다.

그렇기 때문에 외부에서 값을 정하거나 변경하기 위해 멤버 함수를 두는 데 이들을 각 각 getter와 setter라고

부르고 이름 서두에 각각 get과 set을 붙여 쉽게 인식하게 한다.

### 컴파일러가 inline 함수를 인식하는 지점

컴파일러가 inline을 보면 코드를 복사한 뒤에, 컴파일한다.

인라인은 코드를 복사하는 것이기 때문에 전체 코드의 길이는 늘어난다는 단점이 있다.

### 클래스와 구조체의 차이

클래스와 구조체는 완전히 동일하게 작동한다.

유일하게 다른 것은 기본 접근 지정자이다.

클래스에서 기본 접근 지정자는 private이다. 객체를 보호하기 위한 것이다.

반면에, 구조체의 기본 접근 지정자는 public이다. 구조체는 객체를 보호하기 위한 것이 아닌 다양한

타입의 변수들을 한번에 처리 하기 위함이기 때문이다.

### 객체는 실체(instance)

멤버 변수는 상태, 멤버 함수는 행동이라고 부르며

객체는 실행 중에 생성되어 실재하므로 실체라고 부른다.

### 객체의 생성과 소멸

전역 객체는 main() 호출 이전에 생성되고 main() 반환 후에 소멸한다.

함수가 끝날 때 지역 객체는 소멸하고 함수는 반환된다.

만약 객체가 리턴 된다면 객체를 복사한 뒤 원래 객체는 소멸 한 뒤에 복사한 값을 반환한다.

### 객체 배열의 선언은 기본 생성자를 삽입한다.

따라서 매개변수를 받는 생성자만이 존재하고 매개변수가 없는 기본 생성자가 작성되어 있지 않다면,

컴파일 오류가 발생한다.

### 객체 배열은 각 원소마다 생성자와 소멸자가 실행된다.

마찬가지로 소멸할때 높은 인덱스의 객체부터 소멸한다. (생성과 소멸은 스택이기 때문)

### 객체 배열 초기화

Circle circlearray[3] = { Circle a(10, Circle b(20), Circle c() };

배열의 초기화와 마찬가지로 {}로 초기화한다. Circle c는 매개변수 없는 기본 생성자 함수로 생성된다.
