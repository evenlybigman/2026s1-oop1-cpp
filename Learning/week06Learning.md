Circle 실습

- increase 함수로 Circle 객체를 받아 반지름을 키워도 main함수의 객체의 반지름은 커지지않음
- increase 함수에서 매개 변수 객체의 생성자는 호출되지 않았다. 하지만 매개 변수 객체의 소멸자는 호출 되었는데 그것은 호출되는 순간의 실인자 객체 상태를 매개 변수 객체에 그대로 전달하기 위해서 생성자를 호출하지 않은 것이다.

- 이후에 increase 함수에서 매개변수를  포인터 *c로 바꿔서 실행하니 원활하게 반지름이 커졌다.

### 배운점

- 함수에서 매개변수를 클래스 객체로 받을 때 객체 상태를 그대로 전달하기 위해 생성자를 호출하지 않는다.
- 함수에서 다른 함수의 값을 변경하고 싶다면 포인터 변수를 이용한다.

## 5. 4 참조

### & 참조 변수 (별명)

변수에게 별명을 지어줄 수 있음.

참조 변수를 이용하여 선언하면 저장공간을 할당 하지 않고 원래 변수의 공간을 사용함.

예: 

int n = 2;

int& ref = n;

& ref = 3;   <=== ref의 데이터 공간은 존재하지 않고 참조하고 있는 n의 데이터 값이 바뀜 (ref와 n은 완전히 동일한 것)

### 잘 만든 함수란

**한가지를 리턴하는 함수에 두가지 목적을 두지 말 것**

``` C++
#include <iostream>
using namespace std;

int averge(int a[], int size) {
    if (size <= 0) return 0;
    
    int sum = 0;
    for (int i = 0; i < size; i++)
        sum += a[i];
    
    return sum /  size;
}

int main() {
    int x[] = { 1,2,3,4 };
    int avg = average(x, -1);
    if (avg == 0)
        cout << "오류" << endl;
   	else 
        cout << "평균 = " << avg << endl;
}
```

이 코드는 잘 못 만든 함수이다. 평균을 리턴 해주는 averge 함수에 평균을 리턴하는 것과 오류인지 판단하는 것 2개의 목적을 한가지 리턴으로 처리 했기 때문이다. 잘 짠 코드는 1가지 리턴에는 1가지 목적만을 갖는다.

때문에 잘 만든 코드는 다음과 같다.

``` c++
#include <iostream>
using namespace std;

bool average(int a[], int size, int& avg) {
    if (size <= 0) return false;

    int sum = 0;
    for (int i = 0; i < size; i++)
        sum += a[i];

    avg = sum / size;
    return true;
}

int main() {
    int x[] = { 1,2,3,4 };
    int avgg;
    bool success = average(x,3,avgg);
    if (success == false)
        cout << "오류" << endl;
    else
        cout << "평균 = " << avgg << endl;
}
```

이 코드는 평균 값은 avgg에게 주고, 오류 판단은 bool 타입으로 하는 2가지 목적을 2가지 리턴으로 처리한

잘 만든 코드이다.

이를 활용하기 위해서 참조 변수를 배운 것이다.



### 참조 리턴

참조를 리턴 하는것

c언어에서는 값만 리턴 가능

C++에서는 참조를 리턴 할 수도 있음.

참조를 리턴 함으로써 원본을 계속 수정할 수 있게 된다.



### 배운점

&가 주소값이외에 또 다른 용도가 있다는것을 알게 되었고 용도를 알게 되니 *가 왜 역참조인지도 알게 되었다.

코드를 잘 짜는법을 통해서 왜 함수의 매개변수에 받는 값 뿐만 아니라 주는 값까지도 매개변수로 받는지 이해가 잘 안갔는데 이번 수업을 통해 하나의 함수로 여러가지의 목적을 수행할 수 있게 코드를 짜는 방법을 배웠다.
