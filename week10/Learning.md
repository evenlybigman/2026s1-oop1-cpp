# 객체지향언어 10주차

### 상속

자식 클래스가 부모의 클래스를 물려 받는 것.
C++의 3가지 특성 중 캡슐화, 다형성 그리고 마지막으로 상속이 있다. 상속이란 기본 클래스의 속성과 기능을 파생 클래스에게 물려주는 것을 말한다. 기본 클래스를 부모 클래스라고도 부르고, 파생 클래스를 자식 클래스라고도 부른다. 다중 상속을 통해서 코드의 재활용성을 높일 수 있다.

### 장점

관리가 용이함

- 모든 공통된 코드들을 하나 씩 수정할 필요 없이 부모의 코드만 수정하면 되기에 프로그램 관리의 용이하다. 

속도가 빠르다.

- 이미 만들어 둔 클래스의 내용들을 재사용할 수 있기 때문에 코드 작성 속도가 빨라진다.

### 특징

파생 클래스는 기본 클래스의 private 멤버를 접근할 수 없다. 파생 클래스의 객체는 만들어질 때 기본 클래스의 멤버들을 다 갖고 태어난다. 하지만 기본 클래스의 private 멤버에 접근할 수는 없다. 따라서 파생 객체가 자신의 상속 받은 private 멤버를 접근하기 위해서는 상속 받은 기본 클래스의 public 멤버를 이용해서 접근해야 한다.

### 업 캐스팅 (up-casting)

파생 클래스 포인터가 기본 클래스 포인터에 치환되는 것

형 변환을 하지 않아도 컴파일러가 자동으로도 해준다.

### 다운 캐스팅 (down-casting)

기본 클래스 포인터가 파생 클래스 포인터에 치환되는 것

(colorPoint *) &p; 처럼 형 변환을 해주어야 한다.

 ### 접근 지정자 Protected

접근 지정자 protected는 파생 클래스의 객체 내부에서 접근할 수 있게 허용해주는 지정자이다.

### 생성자 함수의 관계

파생 클래스가 생성될 때 파생 클래스의 생성자 함수와 기본 클래스가 모두 생성된다.

### 다중 상속

파생 클래스가 상속을 받을 때 한개의 기본 클래스뿐만이 아니라 여러개의 클래스를 상속 받을 수 있다.

이 때 는 ,를 이용해서 클래스를 나열하면 된다.

### 다중 상속의 문제점

다중 상속을 사용하다가 똑같은 클래스를 두번 상속 받을 수가 있다. 이 때 그 클래스의 멤버들이 2개씩 생기게 된다. 2개가 생겼기 때문에 상속 받은 클래스가 멤버를 사용할 때 멤버가 2개씩 존재하기 때문에 컴파일러가 둘중에 어떤 멤버를 사용하는 것인지 알 수 없기에 모호성이 생기게 된다. 이때 오류가 발생한다.

이를 해결하기 위해 virtual Function를 이용할 수 있다. Virtual 키워드를 사용해 가상 함수를 선언하게 되면 그 객체가 생성될 때 상속 받는 클래스의 중복을 막을 수 있다.
